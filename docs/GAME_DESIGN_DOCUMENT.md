# 游戏设计文档 (GDD) - Fatebound: The Whispering Gods

---

## 1. 游戏总览

### 1.1 游戏名称

**Fatebound: The Whispering Gods**
《命运之缚:低语诸神》

### 1.2 游戏类型

- Roguelike
- RPG
- 叙事驱动
- AI 增强交互剧情

### 1.3 平台

- PC（优先）
- 引擎:Unity / Godot / Unreal（待定）

---

## 2. 核心幻想

> 玩家作为凡人,在一个逐渐崩坏的世界中前行,而诸神与邪灵不断低语,提供力量——但每一份力量都伴随着代价。

主题围绕:

- 诱惑
- 风险
- 信仰
- 命运
- 自由意志 vs 宿命

---

## 3. 核心玩法循环

```
开始一局 Run
→ 探索程序生成地图
→ 战斗遭遇
→ 事件 / 对话房间
→ 与神明或邪灵交流
→ 获得祝福 / 诅咒 / 钥匙
→ 构筑与叙事状态变化
→ Boss 战
→ 结局演算
→ 元进度解锁
```

---

## 4. 玩家系统

### 4.1 玩家基础属性

- 生命值（HP）
- 攻击力（Attack）
- 防御（Defense）
- 暴击率（Crit）
- 腐化值（Corruption）
- 命运值（Fate）
- 神明好感（Favor,按神区分）

### 4.2 元进度系统

局外可解锁:

- 新神明
- 新对话选项
- 新遗物池
- 新结局分支

---

## 5. 神明与实体系统

### 5.1 实体规则结构

```json
{
  "id": "god_war",
  "domain": ["战争", "力量"],
  "tone": "严厉",
  "reward_pool": ["攻击强化", "狂怒模式"],
  "curse_pool": ["生命流失", "敌人强化"],
  "favor_rules": {...}
}
```

### 5.2 好感与敌意

玩家行为会影响:

- 祝福质量
- 诅咒强度
- 对话语气
- 遭遇频率

---

## 6. AI 对话系统

### 6.1 玩家输入 → 意图解析

AI 提取:

- 愿望类型
- 情绪语气
- 风险偏好
- 约束条件

输出示例:

```json
{
  "wish_type": "战斗强化",
  "tone": "绝望",
  "constraints": ["低风险"]
}
```

### 6.2 规则引擎决定结果

**AI 不参与奖励判定**。规则系统负责选择:

- 奖励
- 代价
- 剧情标记

### 6.3 AI 生成神谕叙事

AI 输入包括:

- 神明人格
- 奖励 ID
- 诅咒 ID
- 玩家状态
- 剧情上下文

AI 输出:

- 沉浸式神明回应文本（不允许新增事实）

---

## 14. 风险与对策

| 风险           | 解决方案       |
| -------------- | -------------- |
| AI 幻觉        | 严格 JSON 约束 |
| 平衡崩坏       | 仅规则引擎判定 |
| 成本过高       | AI 仅用于对话  |
| 玩家理解成本高 | UI 明确反馈    |

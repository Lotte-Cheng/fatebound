# Dungeon Runtime Rework

更新时间：2026-02-19

## 重构目标

将原“文本主导 + 高复杂规则结算”重构为“动作房间探索主导”：

1. 角色实时移动与战斗。
2. 清房后通过房门选择路径。
3. 锁门路径需要钥匙，解锁后可进入大宝箱房。
4. 祈福房可请求神像并获得增益。
5. 小地图随探索点亮，并提供有限相邻预判。

## 当前实现（已落地）

## 核心玩法循环

1. 进入房间 -> 敌人刷新（带出生安全距离，避免贴脸/重叠）。
2. 击败敌人 -> 房间清空。
3. 清空后门可通行 -> 选择路径前进。
4. 特定门带锁 -> 消耗钥匙解锁。
5. 祈福房：清怪后靠近神像才能请求，输入时角色冻结。
6. 探索过程中小地图点亮，未探索相邻房间显示类型预判。

## 关键交互规则

1. **锁门一致性**
   - 锁按“边”定义，双向一致。
   - 避免“正向锁、反向不锁”的不合理通行。

2. **祈福触发条件**
   - 当前房间类型为祈福房。
   - 房间已清空。
   - 角色位于神像交互半径内。

3. **输入冻结**
   - 祈福请求输入框聚焦时，角色移动冻结。
   - 避免输入文本时误操作移动与战斗。

## 数据驱动

文件：`data/dungeon_layout.json`

1. 房间图（`x/y` 坐标 + `exits`）。
2. 房间类型（`start/battle/prayer/chest`）。
3. 敌人数量、敌人类型、钥匙奖励。
4. 锁门要求（`requires_key`）。
5. 祈祷池（`prayer_pool`）与大宝箱奖励（`big_reward`）。

## 运行入口

默认启动场景：`res://scenes/DungeonRun.tscn`

## 操作说明

1. `WASD` / 方向键：移动。
2. 鼠标左键：射击。
3. 清房后走到门口：切换房间。
4. 祈福房靠近神像后：输入请求并祈祷。
5. 左侧日志：观察收益、伤害、解锁、祈福等事件。
